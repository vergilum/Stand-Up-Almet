# Стендап Альметьевск - Система регистрации и бронирования

Проект по реализации системы регистрации пользователей и бронирования билетов для сайта Standup Almet с использованием Supabase.

## Обзор проекта

Система позволяет пользователям:
- Регистрироваться и авторизовываться на сайте
- Просматривать и управлять своим профилем
- Бронировать билеты на различные типы шоу
- Просматривать историю своих бронирований
- Отменять бронирования

## Технологии

### Frontend
- Vanilla JavaScript (ES6+)
- HTML5, CSS3
- Supabase JavaScript Client (`@supabase/supabase-js`)

### Backend (Supabase)
- PostgreSQL 17+
- Row Level Security (RLS)
- Database Functions and Triggers
- Auth (Email/Password)

## Установка и настройка

### 1. Требования

- Node.js (для установки зависимостей)
- Доступ к Supabase проекту

### 2. Установка зависимостей

```bash
npm install @supabase/supabase-js
```

### 3. Настройка Supabase

1. Создайте проект в [Supabase](https://supabase.com)
2. Скопируйте `.env.example` в `.env` и заполните реальными значениями:

```env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key-here
```

### 4. Настройка базы данных

Выполните SQL-скрипты из папки `supabase/`:

1. **Миграции**: `supabase/migrations/001_initial_schema.sql`
2. **Политики безопасности**: `supabase/policies/rls_policies.sql`

### 5. Запуск

Откройте `index.html` в браузере или разверните на веб-сервере.

## API Методы

### AuthManager

- `init()` - Инициализация системы аутентификации
- `showAuthModal(type)` - Показ модального окна (login/register)
- `handleLogin(email, password)` - Вход пользователя
- `handleRegister(email, password, fullName, phone)` - Регистрация пользователя
- `handleLogout()` - Выход из системы
- `updateAuthUI(isAuthenticated)` - Обновление интерфейса

### ProfileManager

- `loadUserProfile()` - Загрузка профиля пользователя
- `updateProfile(fullName, phone)` - Обновление профиля
- `loadUserBookings()` - Загрузка истории бронирований
- `createBooking(showType, showDate, ticketCount)` - Создание бронирования
- `cancelBooking(bookingId)` - Отмена бронирования
- `renderProfile(profile)` - Отображение профиля
- `renderBookingsList(bookings)` - Отображение списка бронирований

## База данных

### Таблицы

#### profiles
- `id` (UUID, FK → auth.users) - Идентификатор пользователя
- `full_name` (TEXT) - Полное имя
- `phone` (TEXT) - Телефон
- `created_at` (TIMESTAMP) - Дата создания
- `updated_at` (TIMESTAMP) - Дата обновления

#### bookings
- `id` (UUID) - Идентификатор бронирования
- `user_id` (UUID, FK → profiles) - Пользователь
- `show_type` (TEXT) - Тип шоу (open_mic, best, big)
- `show_date` (DATE) - Дата шоу
- `ticket_count` (INTEGER) - Количество билетов
- `status` (TEXT) - Статус (pending, confirmed, cancelled)
- `created_at` (TIMESTAMP) - Дата создания
- `updated_at` (TIMESTAMP) - Дата обновления

### Безопасность

- **RLS (Row Level Security)**: Пользователи видят только свои данные
- **Автоматическое создание профиля** при регистрации
- **Валидация данных** на уровне базы данных
- **Ограничение количества билетов** (1-10)
