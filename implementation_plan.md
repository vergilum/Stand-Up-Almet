# План реализации системы регистрации пользователей для Standup Almet

## Обзор
Реализация системы регистрации пользователей с возможностью бронирования билетов для сайта Standup Almet с использованием Supabase. Система будет интегрирована прямо в существующий одностраничный сайт без нарушения текущего дизайна.

## Типы данных
Создание типов данных для управления пользователями, профилями и бронированием билетов:

### Пользователь (User)
- `id`: UUID (первичный ключ)
- `email`: string (уникальный, обязательный)
- `created_at`: timestamp
- `updated_at`: timestamp

### Профиль пользователя (Profile)
- `id`: UUID (внешний ключ на auth.users)
- `full_name`: string (необязательный)
- `phone`: string (необязательный)
- `created_at`: timestamp
- `updated_at`: timestamp

### Бронирование билетов (Bookings)
- `id`: UUID (первичный ключ)
- `user_id`: UUID (внешний ключ на profiles)
- `show_type`: enum ('open_mic', 'best', 'big')
- `show_date`: date
- `ticket_count`: integer
- `status`: enum ('pending', 'confirmed', 'cancelled')
- `created_at`: timestamp
- `updated_at`: timestamp

## Файлы
### Новые файлы для создания:
- `js/auth.js` - Модуль аутентификации и управления сессиями
- `js/profile.js` - Модуль управления профилем и бронированием
- `components/auth-modal.html` - HTML-компонент модального окна авторизации
- `components/profile-modal.html` - HTML-компонент модального окна профиля
- `styles/auth.css` - Стили для форм авторизации и профиля

### Существующие файлы для модификации:
- `index.html` - Добавление модальных окон, кнопок входа/регистрации в навигацию
- Добавление кнопки "Профиль" в навигацию для авторизованных пользователей
- Интеграция JavaScript-модулей в существующий скрипт

### Конфигурационные файлы:
- `.env` - Переменные окружения для Supabase (не будет добавлен в репозиторий)

## Функции
### Новые функции:

#### auth.js
- `initSupabase()` - Инициализация клиента Supabase
- `checkAuthState()` - Проверка состояния аутентификации
- `showAuthModal(type)` - Показ модального окна (login/register)
- `handleLogin(email, password)` - Обработка входа пользователя
- `handleRegister(email, password, fullName, phone)` - Обработка регистрации
- `handleLogout()` - Выход из системы
- `updateAuthUI()` - Обновление интерфейса в зависимости от состояния авторизации

#### profile.js
- `loadUserProfile()` - Загрузка данных профиля пользователя
- `updateProfile(fullName, phone)` - Обновление профиля
- `loadUserBookings()` - Загрузка истории бронирования
- `createBooking(showType, showDate, ticketCount)` - Создание бронирования
- `cancelBooking(bookingId)` - Отмена бронирования
- `renderBookingsList()` - Отображение списка бронирований

### Модифицированные функции:
- Существующая функция анимации в `index.html` - добавление логики для скрытия/показа элементов в зависимости от состояния авторизации

## Классы
### Новые CSS-классы для стилей:
- `.auth-modal` - Основной контейнер модального окна авторизации
- `.auth-form` - Стили для форм входа и регистрации
- `.profile-card` - Карточка профиля пользователя
- `.booking-item` - Элемент списка бронирования
- `.user-nav` - Элементы навигации для авторизованных пользователей

### JavaScript-классы:
- `AuthManager` - Класс для управления аутентификацией
- `ProfileManager` - Класс для управления профилем и бронированием
- `ModalManager` - Класс для управления модальными окнами
